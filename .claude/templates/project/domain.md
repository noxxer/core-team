# Знания предметной области

> Факты о реальности: рынок, пользователи, система, процессы, ограничения.
> Это НЕ решения команды (решения → `decisions/`).
> Это то, что мы знаем (или утверждает основатель) о мире, в котором работает проект.

## Отличие от других артефактов

| Артефакт | Тип информации | Пример |
|----------|---------------|--------|
| **domain.md** (этот файл) | Факты о реальности | "85% пользователей — мобильные" |
| decisions/ | Выборы команды | "Используем PostgreSQL" |
| ledger.md | Текущее состояние | "Фаза: MVP, блокер: API" |
| glossary.md | Значения терминов | "'Сессия' = рабочая встреча" |

## Соглашения

- Каждый тезис — утверждение о реальности, проверяемое или опровергаемое
- **Источник** обязателен: основатель, исследование, наблюдение, метрика
- При опровержении — обновить тезис (не удалять, VCS хранит историю)
- Все роли дополняют в своих доменах. Keeper поддерживает структуру
- Группировка по предметным блокам, не хронологически

## Формат тезиса

```
### Блок: [название предметного блока]

- **Тезис:** [утверждение о реальности]
  **Источник:** [основатель | исследование | метрика | наблюдение]
  **Дата:** YYYY-MM-DD
```
