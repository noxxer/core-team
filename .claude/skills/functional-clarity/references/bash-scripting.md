# Bash-scripting — корректные скрипты под текущую среду

> Источник принципов — `i-m-senior-developer` (@spumer, MIT). Расширено под стандарты Core Team.

## Задача

Генерируй bash-скрипты, синтаксически корректные для **той версии bash, что реально стоит в среде** — не для абстрактной «современной».

## Правила

1. **Проверь версию bash перед генерацией.** `bash --version` → используй только возможности, доступные в этой версии. macOS по умолчанию несёт bash 3.2 (2007) — там нет ассоциативных массивов (`declare -A`), `${var,,}`/`${var^^}`, `mapfile`/`readarray`. Если фича нужна — либо проверь наличие, либо деградируй на POSIX-совместимый вариант.

2. **Shebang всегда через окружение:** `#!/usr/bin/env bash`, не `#!/bin/bash`. Это находит bash из PATH (напр. свежий из Homebrew в `/opt/homebrew/bin`), а не системный 3.2.

3. **Fail-fast по умолчанию:** `set -euo pipefail` в начале скрипта (FC принцип fail-fast). `-e` падает на ошибке, `-u` — на необъявленной переменной, `-o pipefail` — на ошибке в любом звене пайпа.

4. **Кавычь расширения переменных:** `"$var"`, `"${arr[@]}"` — иначе слова разъедутся по пробелам и глоб раскроется неожиданно.

5. **Очистка через `trap`:** временные файлы (`mktemp`) убирай в `trap 'rm -f "$tmp"' EXIT` — контекстный аналог `finally`, состояние не течёт.

6. **Проверяй внешние зависимости явно:** `command -v curl >/dev/null 2>&1 || { echo "нужен curl" >&2; exit 1; }` — fail-fast с человекочитаемым сообщением вместо невнятного падения посреди скрипта.

## Анти-паттерны

- Bash 4+ синтаксис без проверки версии → скрипт падает на macOS-дефолте.
- `#!/bin/bash` хардкодом → игнорирует обновлённый bash из PATH.
- Молчаливое продолжение после ошибки (`cmd || true` без причины) → Error Hiding.
- Некавыченный `$(...)` в условиях → словоделение ломает логику.
